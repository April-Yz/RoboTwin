# Task: 为 OpenVLA-OFT 增加 Offline Privileged Self-Distillation
请先阅读现有训练入口、dataset、model wrapper、action head 相关代码，再按“最小侵入式修改”原则实现本需求。优先保证兼容原训练流程，不要先大规模重构。

## 1. 目标

在现有 OpenVLA-OFT 代码基础上，实现一个新的 **offline privileged self-distillation** 训练模式。

核心要求：

- **student** 只使用当前时刻可部署输入
- **teacher** 使用与 student 相同的主干结构，但额外看到来自 demo 的未来信息
- 训练方式仍然是 **offline**
- 不接入 simulator
- 不做 on-policy rollout
- teacher 只提供监督，不参与梯度更新
- 第一版先实现最小可行版本：
  - teacher 多看未来图像信息
  - 蒸馏目标先做 action-level distillation

---

## 2. 方法定义

### Student 输入
student 只能看到当前信息：

- `curr_image`
- `curr_wrist_image`（如果项目已有）
- `curr_state`（如果项目已有）
- `instruction`

### Teacher 输入
teacher 使用 student 的全部输入，并额外看到：

- `future_images = [o_{t+1}, ..., o_{t+k}]`

未来信息先编码为一个 `future_latent`，再注入 teacher 分支。

### Teacher / Student 关系
- teacher 和 student 使用相同 backbone
- 训练时允许同一个模型做两次 forward：
  - 一次 student forward
  - 一次 teacher forward
- teacher forward 默认在 `torch.no_grad()` 下执行
- teacher 不更新参数

## 3. 本轮实现范围

### 必做
1. dataset 返回未来图像窗口
2. 新增 future encoder / projector
3. 新增 student forward 和 teacher forward
4. 新增蒸馏损失
5. 将蒸馏损失与原 BC loss 组合
6. 通过配置项控制是否启用
7. 不启用时保持原训练逻辑不变

### 第一版不做
- on-policy rollout
- simulator 训练
- EMA teacher
- 多层特征蒸馏
- attention 蒸馏
- future action 输入
- RL / policy gradient

## 4. 新增配置项

请新增以下配置项，命名可微调，但语义必须保留：

```yaml
use_privileged_distill: true
future_horizon: 4
future_mode: image_latent
distill_target: action
distill_loss_type: mse
distill_weight: 0.5
bc_weight: 1.0
teacher_detach: true

说明：

use_privileged_distill：是否启用 teacher-student 蒸馏

future_horizon：teacher 可看的未来步数

future_mode：第一版只实现 image_latent

distill_target：第一版只实现 action

distill_loss_type：第一版先实现 mse

distill_weight：蒸馏损失权重

bc_weight：原 BC 损失权重

teacher_detach：teacher 是否 stop-grad / no-grad

5. 数据集改动

当 use_privileged_distill = true 时，dataset 需要额外返回：

future_images

可选 future_mask

推荐 sample 结构：

sample = {
    "curr_image": ...,
    "curr_wrist_image": ...,
    "curr_state": ...,
    "instruction": ...,
    "action": ...,
    "future_images": ...,
    "future_mask": ...,
}

规则：

对每个时间步 t，未来窗口取 [t+1, ..., t+k]

如果超出轨迹末尾：

允许 repeat 最后一帧

或返回 future_mask

batch 内张量尺寸必须一致

限制：

第一版不要把 future action 作为 teacher 输入

student 不能使用任何 future 信息

6. 模型改动
6.1 新增 future encoder / projector

新增一个轻量模块，把 future_images 编码为 future_latent。

允许的实现方式：

复用已有视觉编码器提特征，再做时间维 pooling

或实现一个轻量 future projector

输出目标：

一个固定维度的 future_latent

6.2 teacher 注入方式

teacher 需要接收 future_latent。

第一版允许以下任一实现方式：

作为额外 token 拼接到输入

作为额外条件输入 action head

通过 FiLM / adaptor 注入（若原项目已有接口）

要求：

优先最小侵入式修改

尽量不要重构 backbone 主体

6.3 前向接口

新增清晰的 teacher/student 前向语义，例如：

student_out = model.forward_student(batch)
teacher_out = model.forward_teacher(batch)

要求：

forward_student 不看 future 信息

forward_teacher 使用 future 信息

teacher 分支默认 no_grad

7. Loss 改动

保留原始 BC / action prediction loss，记为：

loss_bc

新增蒸馏损失：

loss_distill = MSE(student_action, teacher_action)

总损失：

loss = bc_weight * loss_bc + distill_weight * loss_distill

要求：

不启用 use_privileged_distill 时，行为退化为原始训练逻辑

日志中至少记录：

loss_total

loss_bc

loss_distill

8. 训练流程

当 use_privileged_distill = false 时：

保持原训练流程不变

当 use_privileged_distill = true 时，每个 step 执行：

从 dataloader 读取 batch

构造 student 输入

student forward，得到 student_action

在 torch.no_grad() 下执行 teacher forward，得到 teacher_action

计算 loss_bc

计算 loss_distill

计算总损失并反向传播

仅更新 student 可训练参数

9. 推荐代码组织

优先采用最小侵入式修改。

建议新增：

future_projector.py

distill_wrapper.py

distill_losses.py

建议写一个 wrapper，例如：

class PrivilegedDistillWrapper(nn.Module):
    def __init__(self, ...):
        ...
    def forward_student(self, batch):
        ...
    def forward_teacher(self, batch):
        ...

这样尽量少改原 backbone。
