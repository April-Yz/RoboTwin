# V1.5 AnyGrasp Dual-Arm Synchronized Stage Execution

Date: 2026-03-17

## What changed
- Added CLI arg `--execute_both_arms` in:
  - `code_painting/plan_anygrasp_keyframes_r1.py`
  - `code_painting/plan_anygrasp_keyframes_r1_batch.py`
- When `--arm auto --execute_both_arms 1`:
  - planner still performs candidate selection as before,
  - then enters synchronized dual-arm stage control.
  - in each stage, both arms are planned and executed together, and stage transition happens only when both arms reach tolerance.

## Why
Users needed one run that verifies target reach for both left and right hands, instead of only selected-arm execution plus non-actuated supervision.

## Execution logic
Dual-arm stage loop:
1. Open both grippers in init.
2. Stage `pregrasp`: repeatedly plan left/right and execute both trajectories together until both reached or attempt budget exhausted.
3. Stage `grasp`: same dual-arm synchronized strategy.
4. Close both grippers.
5. Stage `action`: same dual-arm synchronized strategy.
6. Record per-attempt dual-arm history and per-arm reached metrics.

## Summary fields
New fields:
- `execute_both_arms`
- `stages_by_executed_arm`
- `selected_objects_by_executed_arm`
- `selected_candidates_by_executed_arm`
- `supervision_targets_by_executed_arm`
- `supervision_only_arms_by_executed_arm`

Backward compatibility:
- Existing `stages` and `selected_candidates` remain and refer to the primary executed arm.

## Notes
- This is synchronized dual-arm stage execution driven by per-arm planners; trajectories are applied in lockstep per step.
- For consistent two-arm behavior, provide valid manual candidates for both arms.
