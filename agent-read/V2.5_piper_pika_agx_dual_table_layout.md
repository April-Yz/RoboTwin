# V2.5 Piper+Pika (agx) Dual Table Layout

## Goal

Create a UR-style dual-single-arm layout using the newer colored Piper+Pika combo and place both robots along the long edge of a table.

## Table geometry

Requested table dimensions:

- height: `0.75 m`
- width: `0.60 m`
- length: `1.20 m`

In the preview scene, the table is modeled as a simple box:

- center: `[0.0, 0.0, 0.375]`
- half size: `[0.60, 0.30, 0.375]`

So the tabletop plane is at:

- `z = 0.75`

## Layout assumption used

To stay consistent with the earlier UR-style split approach, a symmetric dual-instance layout was used.

Shared pose before split:

- previous incorrect version: `robot_pose = [0.0, -0.60, 0.75, 1.0, 0.0, 0.0, 0.0]`
- orientation-corrected but still table-outside version: `robot_pose = [0.0, -0.60, 0.75, 0.70710678, 0.0, 0.0, 0.70710678]`
- current edge-mounted version: `robot_pose = [0.0, -0.30, 0.75, 0.70710678, 0.0, 0.0, 0.70710678]`

Split distance:

- `embodiment_dis = 0.60`

Effective bases after `Robot(...)` applies the left/right split:

- left base: `[-0.30, -0.60, 0.75]`
- right base: `[+0.30, -0.60, 0.75]`

Interpretation:

- both robots are placed along the same long edge of the table
- the table edge is at `y = -0.30`
- current robot bases are at `y = -0.30`
- therefore the current version is edge-mounted on the tabletop rather than outside the table
- the left base is `0.30 m` away from the left short-side end
- the right base is `0.30 m` away from the right short-side end
- the base-to-base spacing is therefore `0.60 m`
- after the quaternion correction, both robot fronts point toward `+y`, i.e. from the long-edge outside toward the tabletop interior
- this matches the intended use: grippers approach and operate objects on the tabletop from one long-edge side

## Camera note

The earlier oblique preview was effectively using a third-person style camera, and the image orientation looked flipped (ceiling/floor sense inverted). For temporary inspection, a new top-down camera mode was added.

Top-down camera placement:

- centered between the two robot bases
- world position: `[0.0, -0.60, 2.10]`
- looks straight down toward the tabletop

Reference from the existing UR-style replay logic:

- `code_painting/replay_piper_dual_h5.py` uses a fixed overview/head camera fallback:
  - position: `[-0.032, -0.45, 1.35]`
  - forward: `[0.0, 0.6, -0.8]`
  - left: `[-1.0, 0.0, 0.0]`
- `code_painting/render_hand_retarget_r1_npz.py` builds the third-person camera from robot forward direction and world-up, rather than using an arbitrary flipped camera

Based on that reference, the oblique preview here was corrected to sit behind the robots and look toward the tabletop from the long-edge side.

This is now available through:

- `--camera-mode top_down`
- `--camera-mode oblique`
