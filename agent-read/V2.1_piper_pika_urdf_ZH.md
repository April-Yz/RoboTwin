## 颜色来源检查

### Piper 手臂

原始 Piper 手臂的颜色信息，大概率是在以下 DAE 文件中：

- `/home/zaijia001/Downloads/agx_arm_urdf/piper/meshes/dae/`

证据：

- `link1.dae` 以及其他 arm DAE 文件中都能看到 Collada 材质定义，例如：
  - `<library_effects>`
  - `<library_materials>`
  - `<diffuse><color ...>`

所以对手臂来说，原始外观颜色主要应来自 DAE visual mesh，而不是 STL。

### Pika 夹爪

在当前检查到的 Pika gripper 源代码树中，没有发现 DAE / OBJ / MTL / 纹理图片这类带更丰富颜色信息的资源。

当前 gripper package 主要提供的是：

- STL mesh：
  - `gripper_base_link.STL`
  - `gripper_left_link.STL`
  - `gripper_right_link.STL`
- URDF 中直接写死的白色材质：
  - `rgba="1 1 1 1"`

因此，基于当前已检查的源码树，Pika 原始包本身看起来并没有附带更丰富的 mesh 颜色信息；它的可见颜色大概率就是依赖 URDF material 设置。

## 手臂 visual 切回 DAE

为了尽量恢复 Piper 手臂原始外观，这一轮已经把 `piper_pika.urdf` 更新为：

- 手臂 visual mesh 改为 DAE：
  - `meshes/dae/base_link.dae`
  - `meshes/dae/link1.dae`
  - `meshes/dae/link2.dae`
  - `meshes/dae/link3.dae`
  - `meshes/dae/link4.dae`
  - `meshes/dae/link5.dae`
  - `meshes/dae/link6.dae`
- collision mesh 仍然保留 STL
- Pika 夹爪仍使用 STL，因为在当前检查到的源 package 中没有发现更适合保留原始颜色的 mesh 格式

## 当前组合模型状态

当前组装后的模型已经是你要的这类组合：

- Piper 手臂：
  - visual 使用原始 Piper 的 DAE 资源
  - 目标是尽量保留 Piper 手臂原始颜色
- Pika 夹爪：
  - visual 仍然使用原始 Pika 的 STL 资源
  - 可见颜色仍来自 Pika 原始包里写死的白色 URDF material

所以当前 `piper_pika.urdf` 可以理解为：

- 一个带原始材质外观的 Piper 手臂
- 接上一个无额外颜色资源、保持白色的 Pika 夹爪
