# V1.7 物体流程

## 范围

这份文档说明 RoboTwin 这一侧的 FoundationPose 物体 replay 流程。

边界：
- FoundationPose 提取是上游步骤
- RoboTwin 负责消费 `poses.npz`
- RoboTwin 当前不负责估计 FoundationPose

- 多物体批处理 replay: [run_multi_object_pose_r1_npz_batch.sh](/home/zaijia001/ssd/RoboTwin/code_painting/run_multi_object_pose_r1_npz_batch.sh)

## 输入与输出

### 输入

- FoundationPose `poses.npz`
- 物体 mesh `.obj`

典型多物体输入根目录：
- `/home/zaijia001/ssd/data/R1/gt_depth_vis/<task>_multi/obj_vis`

例子：

```text
obj_vis/
  d_pour_blue_1/
    bottle/poses.npz
    cup/poses.npz
```

### 输出

- `head_cam_replay.mp4`
- 可选 `third_cam_replay.mp4`
- 可选 `head_depth_frames/`
- 可选 `head_anygrasp_frames/`
- `object_world_poses.npz`
- `multi_object_world_poses.npz`

## 逻辑

1. 读取 FoundationPose 在相机系下的物体姿态
2. 构建 RoboTwin replay renderer
3. 把相机系姿态转成 RoboTwin 世界系姿态
4. 渲染 replay 视频
5. 可选导出深度图和 AnyGrasp RGB-D

## 关键参数

### 数据选择

- `--input_npz`：单物体 `poses.npz`
- `--input_root`：多物体根目录
- `--ids`：只跑部分视频 id
- `--objects`：只跑部分物体

### 回放范围

- `--frame_start` `--frame_end` `--frame_stride` `--max_frames`
- `--missing_frame_policy hide|hold_last`

### 输出

- `--head_only`：只保留头相机
- `--save_png_frames`：保存 RGB 帧
- `--save_head_depth`：保存 uint16 depth PNG
- `--save_anygrasp_frames`：导出 AnyGrasp RGB-D

### 场景可视化

- `--hide_robot`：保留相机位姿但隐藏机器人
- `--disable_table`：隐藏桌子
- `--lighting_mode default|front|front_no_shadow`：灯光模式

## 为什么这一段重要

这一步是 FoundationPose 到 AnyGrasp 的桥梁：

- FoundationPose 提供相机系物体姿态
- RoboTwin 把它变成世界系轨迹
- RoboTwin 再导出 RGB-D 和相机内参供 AnyGrasp 使用
