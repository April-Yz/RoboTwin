# V2.4 Piper 颜色诊断：DAE 灰色 vs URDF 覆盖

## 用户当前观察到的稳定现象

用户当前观察结果是稳定的：

- 原始 Pika gripper 单独显示：白色
- 原始 Piper 手臂单独显示：灰色
- 组合后的 `piper_pika`：发白 / 被洗浅

这一轮不做代码修改，只做结论整理与记录。

## 诊断结论

是的——目前证据已经很支持下面这个判断：

- 原始 Piper 手臂的灰色，来自 Piper 自身的 DAE 文件材质
- 当 Piper 被放进组合后的 `piper_pika.urdf` 里时，这个来自 DAE 的灰色外观，很可能会被组合 URDF 中仍然保留的显式材质颜色覆盖或冲淡

可以直接总结成：

- **原始 Piper 的灰色 = 来自 DAE**
- **组合后 Piper 发白 = 很可能被 `piper_pika.urdf` 里的 URDF material 覆盖**

## 证据来源

### 1. Piper 的 DAE 内部确实带有深灰材质

示例文件：

- `/home/zaijia001/Downloads/agx_arm_urdf/piper/meshes/dae/link1.dae`

其中 Collada 材质包含：

```xml
<diffuse>
  <color sid="diffuse">0.113725 0.113725 0.113725 1</color>
</diffuse>
```

这大约对应：

- RGB ≈ `(29, 29, 29)`（0–255 量级）

所以原始源资产本身确实是深灰色，不是白色。

### 2. 组合版 URDF 在 Piper 手臂 link 上仍然保留了显式浅色 material

文件：

- `/home/zaijia001/ssd/RoboTwin/assets/embodiments/piper_pika/piper_pika.urdf`

其中多个 Piper 手臂 link 都有类似：

```xml
<material>
  <color rgba="0.792156862745098 0.819607843137255 0.933333333333333 1" />
</material>
```

这大约对应：

- RGB ≈ `(202, 209, 238)`

这是一个非常浅的偏蓝灰色。

所以当前组合模型实际上是同时存在：

- Piper 的 DAE visual mesh
- 以及从组合参考里继承下来的显式浅色 URDF material

这就是为什么组合后的 Piper 手臂不再像原始 Piper 那样保持深灰的最主要原因。

## 如何理解这些比较结果

1. 对原始单独 Piper 来说，暗灯光版本比亮灯光版本更接近 DAE 里的深灰参考值。
2. 但即使在暗灯光下，组合模型仍然明显比原始单独 Piper 更远离 DAE 参考值。
3. 这说明组合发白并不只是灯光太亮。
4. 当前最强的解释仍然是：组合 URDF 中显式的浅色 material 覆盖了 Piper DAE 的原始灰色外观。

## 这些图像统计的局限

上面的 PNG 颜色统计只是近似值，因为它会受到这些因素影响：

- 灯光方向和强度
- 阴影
- 相机响应 / tone mapping
- 背景和分割误差
- 渲染器对高光和明暗的处理

所以真正最可信的“颜色真值”仍然是：

- DAE 文件内部的材质数值
- 以及 URDF 里可能覆盖它的 material 数值

## 有什么方法可以尽量无损地可视化 Piper 原始色彩？

严格来说，只要是带光照的 3D 渲染，就不可能对“源材质颜色”做到完全无损，因为最终像素一定会被光照和明暗影响。

但更接近“无损查看原始颜色”的做法有下面几种：

### 方案 A——把 DAE 材质数值当作真值
直接读取 DAE 里的 `<diffuse><color>` 数值。

对于颜色诊断来说，这是最接近无损的源颜色检查方式。

### 方案 B——做 flat diagnostic render
做一种诊断式渲染，尽量满足：

- 不使用 URDF 的额外 material 覆盖
- 中性背景
- 不使用 HDRI / 环境贴图
- 只用最小单白光，或者直接用 unlit / flat shading
- 关闭 tone mapping / 自动曝光类影响

这样得到的视觉效果会更接近原始材质颜色。

### 方案 C——把原始 DAE 放进 Blender 查看
如果目标是做最可控的人眼检查，通常最好的方式是：

- 把原始 Piper DAE 导入 Blender
- 用平坦 / 材质预览模式查看
- 关闭过强环境光
- 使用 Standard color management 和固定曝光

这通常是最干净、最稳定的人眼确认流程。

## 如果后续要保留准确组合外观
如果以后要让组合机器人同时满足：

- Piper 保持原始灰色
- Pika 保持白色

那么最可能正确的修复方式是：

- 删除 `piper_pika.urdf` 中 Piper 手臂各 link 上显式的浅色 `<material><color ...>`
- 保留 Pika 夹爪的白色 material
- 再重新渲染对比
