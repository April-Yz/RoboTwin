# V1.7 手部流程

## 范围

这份文档只说明 RoboTwin 这一侧的人手流程。

边界：
- `hand_detections_<id>.npz` 被视作上游输入
- RoboTwin 负责 replay
- RoboTwin 当前不负责生成这个文件

## 主要脚本

- 基础 replay: [run_hand_retarget_r1_npz.sh](/home/zaijia001/ssd/RoboTwin/code_painting/run_hand_retarget_r1_npz.sh)
- URDFIK replay: [run_hand_retarget_r1_npz_urdfik.sh](/home/zaijia001/ssd/RoboTwin/code_painting/run_hand_retarget_r1_npz_urdfik.sh)
- 批处理 replay: [run_hand_retarget_r1_npz_urdfik_pool.sh](/home/zaijia001/ssd/RoboTwin/code_painting/run_hand_retarget_r1_npz_urdfik_pool.sh)

## 输入与输出

输入：
- `hand_detections_<id>.npz`

典型输入路径：
- `/home/zaijia001/ssd/data/R1/gt_depth_vis/<task>/hand_vis/hand_detections_<id>.npz`

典型输出目录：
- `/home/zaijia001/ssd/RoboTwin/code_painting/output_hand_retarget_*`

## 逻辑

1. 读取手部 NPZ
2. 根据 stored gripper pose 或 keypoints 构造左右手 target pose
3. 转成 RoboTwin 使用的坐标系
4. 在仿真中规划并重放
5. 保存视频和 debug 结果

## 关键参数

### 输入与帧选择

- `--input_npz`：输入 NPZ
- `--frame_start` `--frame_end` `--frame_stride` `--max_frames`：帧截取
- `--arms left|right|both`：选择执行手

### 目标构造

- `--pose_source gripper|retreat`：使用 gripper pose 还是 retreat pose
- `--require_stored_gripper_pose 1`：要求 NPZ 已经带有 gripper pose
- `--retreat_distance`：retreat 偏移

### 朝向

- `--orientation_remap_label`：固定坐标轴 remap
- `--stored_orientation_post_rot_xyz_deg`：固定后置欧拉角旋转
- `--debug_force_orientation`：强制用某种 debug 朝向替代

### 位置修正

- `--target_world_offset_xyz`：全局 XYZ 偏移
- `--left_target_world_offset_xyz`：左手单独偏移
- `--right_target_world_offset_xyz`：右手单独偏移
- `--target_world_z_offset`：全局 Z 偏移

### 可视化

- `--enable_viewer`：打开 SAPIEN viewer
- `--viewer_wait_at_end`：结束后保持窗口
- `--debug_visualize_targets`：显示目标坐标轴
- `--lighting_mode default|front|front_no_shadow`：灯光模式

  --viewer_wait_at_end 1

- 想用当前推荐配置：用 `run_hand_retarget_r1_npz_urdfik.sh`
- 想整批跑数据集：用 `run_hand_retarget_r1_npz_urdfik_pool.sh`
