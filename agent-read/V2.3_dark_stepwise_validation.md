# V2.3 Dark Stepwise Validation for Piper / Pika / Combined Model

## Goal

Validate appearance step by step under darker lighting:

1. original Piper arm alone
2. original Pika gripper alone
3. combined `piper_pika` model

## Main conclusion

The whitening of the combined model is not only a lighting issue.

A structural difference was identified:

- original Piper arm URDF:
  - uses DAE visual meshes
  - does **not** define per-link URDF `<material><color ...>` blocks for the arm links
  - therefore SAPIEN can rely on the DAE-embedded material/effect information
- combined `piper_pika.urdf`:
  - arm links now point to DAE visual meshes
  - but still keep explicit URDF material colors inherited from the previous combined reference, e.g.
    - `rgba="0.792156862745098 0.819607843137255 0.933333333333333 1"`
  - this likely overrides or washes out the original DAE material appearance in SAPIEN

So the current best explanation is:

- original Piper arm can show its intended darker tone
- combined arm becomes pale because the combined URDF still contains explicit light-colored material overrides on the Piper arm links
