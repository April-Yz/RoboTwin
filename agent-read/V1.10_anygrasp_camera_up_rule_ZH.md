# V1.10 AnyGrasp 相机朝上规则

## 这次改了什么

给 AnyGrasp candidate 加了一层 planner 侧姿态后处理：

- `--candidate_keep_camera_up 1`
- `--candidate_camera_top_axis z|y`

打开后，planner 会去掉夹爪绕前进轴的自由自转，并把指定的局部“顶部轴”尽量对齐到世界坐标的 `+Z`。

## 为什么要加这个规则

这次用户观察到：

- keyframe 1 和 keyframe 22 的夹爪姿态，左手看起来差不多绕自己转了 `360` 度
- 但对抓取来说，这两种姿态其实等价

这说明当前姿态里有一维是冗余自由度：

- TCP 位置没变
- 前进/接近方向没变
- 夹爪开口含义没变
- 只是绕前进轴自转了

这对抓取本身通常不敏感，但会影响：

- 夹爪顶部朝哪里
- 夹爪上方安装的小相机朝哪里

用户希望最终行为是：

- 相机始终在上方
- 不允许夹爪绕前进轴随意自转

## 哪一轴是这个“360 度等价”的来源

当前 planner 和机器人链路都表明：

- gripper 的局部 `+X` 就是前进轴 / 接近轴

代码证据：

- [`code_painting/plan_anygrasp_keyframes_r1.py`](/home/zaijia001/ssd/RoboTwin/code_painting/plan_anygrasp_keyframes_r1.py) 里的 `pose_with_offset_along_local_x(...)`
- [`envs/robot/robot.py`](/home/zaijia001/ssd/RoboTwin/envs/robot/robot.py) 里的 `_trans_from_gripper_to_endlink(...)`

所以真正冗余的那一维就是：

- 绕局部 `+X` 的 roll

## 新规则具体怎么做

对每个 AnyGrasp candidate：

1. 先像以前一样，把 camera-space 姿态转成 world-space
2. 如果开了 `--candidate_keep_camera_up 1`：
   - 保持局部 `+X` 不变
   - 把世界上方 `(0, 0, 1)` 投影到垂直于局部 `+X` 的平面里
   - 把这个投影方向当成“目标顶部轴”
   - 重新构造剩下两个正交轴，保持右手坐标系

这样得到的效果是：

- 接近方向不变
- 只消掉冗余 roll
- 夹爪顶部 / 相机尽量保持朝上

如果当前前进轴几乎已经和世界上方平行，那么这个投影会退化，此时保留原始姿态不动。

## 新参数

- `--candidate_keep_camera_up 1`
  - 打开“去掉绕前进轴自由滚转”的规则
- `--candidate_camera_top_axis z`
  - 把局部 `+Z` 当成夹爪顶部 / 相机朝上方向
- `--candidate_camera_top_axis y`
  - 如果后续确认相机安装方向其实对应局部 `+Y`，就改用这个

当前默认建议先用：

- `--candidate_camera_top_axis z`

## 结果里怎么检查

`plan_summary.json` 现在会额外保存：

- `candidate_keep_camera_up`
- `candidate_camera_top_axis`
- `top_axis_up_dot`

其中 `top_axis_up_dot` 表示“顶部轴”和世界 `+Z` 的点积：

- 接近 `1.0`：顶部朝上
- 接近 `0.0`：顶部水平
- 小于 `0`：顶部朝下

## 你前面一次改动了什么，和这次成功有什么区别

最近和这条链路直接相关的改动可以分成两类。

真正改变行为的：

- `d877396`
  - 在 `action` 阶段回放物体轨迹
- `830f057`
  - 支持给 replay 物体替换 mesh，比如换成 `blue_cup.obj`

只用于 debug 的：

- `27a4bc9`
  - 导出 replay 侧的 pose debug
- `6a0974f`
  - 导出 planner 侧的 pose debug，并比较“导出 world pose 时的 head cam”和“当前 planner 的 head cam”

所以如果这次运行变正确了，直接影响行为的主要是：

- action 阶段的物体 replay
- mesh override

后两个 pose debug 提交本身主要是为了排查，不是为了改变 motion logic。
