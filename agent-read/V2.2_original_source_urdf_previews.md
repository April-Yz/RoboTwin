Render the original source URDFs directly from the download folders so their appearance can be compared before further assembly changes.

## Preview script

Added:

- `code_painting/visualize_original_source_urdfs.py`

What it does:

- rewrites `package://...` mesh URIs into absolute local filesystem paths in a temporary URDF
- loads the original source URDF in SAPIEN
- exports one image and one short mp4 for each target

## Output files

Generated under:

- `code_painting/output_original_source_urdf_preview/`

Files:

- `piper_arm.png`
- `piper_arm.mp4`
- `pika_gripper.png`
- `pika_gripper.mp4`

## Validation command

```bash
/home/zaijia001/ssd/miniconda3/envs/RoboTwin_bw/bin/python \
  /home/zaijia001/ssd/RoboTwin/code_painting/visualize_original_source_urdfs.py \
```

## Why this matters

These previews let us compare:

- whether the original Piper arm DAE-based source already looks white/pink in SAPIEN
- whether the original Pika gripper URDF itself is simply white due to its URDF material settings

This helps separate:
- assembly mistakes
from
- original source appearance limitations.
