# V1.7 AnyGrasp 流程

## 范围

这份文档说明 AnyGrasp 结果已经在别的服务器上跑完之后，RoboTwin 这一侧怎么继续做候选选择、可视化和关键帧执行。

边界：
- 这套流程默认不在 RoboTwin 里跑 AnyGrasp 推理
- RoboTwin 负责消费 AnyGrasp 结果并做执行

## 输入

每个视频 id 需要三类输入：

1. AnyGrasp 结果
   - `grasps/grasp_<frame>.json`
2. 物体 replay 结果
   - `multi_object_world_poses.npz`
3. 手部结果
   - `hand_detections_<id>.npz`

典型根目录：

- AnyGrasp：
  - `/home/zaijia001/ssd/RoboTwin/code_painting/anygrasp_batch_results`
- Replay：
  - `/home/zaijia001/ssd/RoboTwin/code_painting/replay_m_obj_pose_d_pour_blue`
- Hand：
  - `/home/zaijia001/ssd/data/R1/gt_depth_vis/d_pour_blue/hand_vis`

## 主要脚本

- 批处理入口: [run_plan_anygrasp_keyframes_r1_batch.sh](/home/zaijia001/ssd/RoboTwin/code_painting/run_plan_anygrasp_keyframes_r1_batch.sh)
- 批处理调度: [plan_anygrasp_keyframes_r1_batch.py](/home/zaijia001/ssd/RoboTwin/code_painting/plan_anygrasp_keyframes_r1_batch.py)
- 单视频实现: [plan_anygrasp_keyframes_r1.py](/home/zaijia001/ssd/RoboTwin/code_painting/plan_anygrasp_keyframes_r1.py)

## 逻辑

1. 读取两个关键帧的抓取候选
2. 读取 `hand_detections_<id>.npz` 里的手部朝向
3. 给左右手候选分别排序
4. 可选加上物体约束和距离约束
5. 可选接受手工指定候选
6. 在 RoboTwin 里执行 `pregrasp -> grasp -> action`
7. 保存视频、总结和 rank preview 图片

## 输出

单视频输出：

- `debug_selection_preview.mp4`
- `debug_execution_preview.mp4`
- `head_cam_plan.mp4`
- `rank_previews/keyframe_<frame>_rank_<k>.png`
- `plan_summary.json`

批处理输出：

- `batch_plan_summary.json`

## 关键参数

### 关键帧和执行手

- `--keyframes A B`：抓取关键帧和操作关键帧
- `--arm auto|left|right`：选择执行手
- `--execute_both_arms 1`：同步双手执行

### 候选筛选

- `--left_target_object`
- `--right_target_object`
- `--candidate_object_max_distance_m`
- `--enforce_target_object_constraint 0|1`
- `--enforce_candidate_distance_constraint 0|1`

### 候选显示和手工指定

- `--debug_candidate_top_k`
- `--debug_common_candidate_top_k`
- `--debug_show_all_candidates 0|1`
- `--manual_candidate FRAME ARM CANDIDATE_IDX`

### 朝向调试

- `--candidate_orientation_remap_label`
- `--candidate_post_rot_xyz_deg`

### 执行精度调试

- `--planner_backend urdfik|curobo`
- `--replan_until_reached 1`
- `--replan_until_reached_max_attempts`
- `--reach_error_pose_source tcp|ee`
- `--init_prefix_frames`

### 可视化

- `--save_debug_preview`
- `--save_debug_execution_preview`
- `--save_rank_preview_images`
- `--rank_preview_top_n`
- `--enable_viewer`
- `--viewer_wait_at_end`

## 可视化语义

- 绿色夹爪：common candidate
- 蓝色夹爪：左手 top 候选
- 橙色夹爪：右手 top 候选
- 红色夹爪：最终选中的候选

候选编号会显示成小号纯数字，不带背景框。

## 怎么使用这些输出

- `debug_selection_preview.mp4`：判断候选是否选得合理
- `rank_previews/`：精确挑选 candidate id
- `debug_execution_preview.mp4`：判断机器人是否先到关键帧 A 再到关键帧 B
- `plan_summary.json`：查看选中的候选、阶段结果和每次 attempt 的误差历史
