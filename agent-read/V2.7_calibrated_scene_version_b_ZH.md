# V2.7 标定场景 Version B

## 目标

采用版本 B 对齐方式：

- 去掉第一台机械臂原先手动加上的 `+90°` 锚定旋转
- 尽量让标定 bundle 里的左右分离继续表现为 world `y` 方向关系
- 不再旋转锚定机械臂，而是把桌子的约定旋转 90°
- 输出统一放在 `code_painting/pika/` 下

## 为什么要做 Version B

之前的版本给第一台锚定机械臂手动加了一个四元数：

- `[0.70710678, 0.0, 0.0, 0.70710678]`

它等价于绕 `z` 轴旋转 `+90°`，这样会把标定 bundle 中本来属于 `left_base` 内在坐标系里的 `y` 向左右分离，映射成 world `x` 方向。

Version B 去掉了这个锚定旋转，改成通过旋转桌子的约定来适配场景。

## 与 R1 / UR 直觉的对齐

这个版本更接近现有 R1 风格直觉：

- 左右分离主要视为 `y` 轴关系
- 第一台机械臂保持 world 单位朝向
- 桌子的长边视为沿 world `y` 方向
- 两个机械臂朝向 world `+x`

## 从 `robot_config_PiperPika_agx_dual_table.json` 中复用了什么，哪些没复用

### 复用的内容

- 原先桌边内缩距离的绝对值：
  - 当前配置里是 `y = -0.25`
  - 在 version B 里复用 `|y| = 0.25` 作为新的 world `x` 内缩量
- 桌面高度：
  - `z = 0.75`

### 没复用 / 有意忽略的内容

- `robot_pose` 里的旧四元数
  - 因为 version B 要去掉之前那个 `+90°` 锚定旋转
- `embodiment_dis`
  - 在这个标定场景脚本里不再用于第二台机械臂摆放
  - 第二台机械臂直接由 `left_base_T_right_base` 决定
- `right_embodiment_config`
  - 在这个标定场景脚本里不使用
  - 第二台机械臂的位置直接由标定 bundle 决定，而不是靠双实例对称拆分

## Version B 的锚定构造方式

结合当前配置和 bundle：

- 左锚定 world `x` = `-abs(config.left_embodiment_config.robot_pose[0][1])`
- 左锚定 world `y` = `-0.5 * left_base_T_right_base.translation.y`
- 左锚定 world `z` = config 中桌面高度
- 左锚定 quaternion = 单位四元数

本次验证得到：

### 左 base
- 位置：`[-0.25, 0.285920, 0.75]`
- 四元数（wxyz）：`[1.0, 0.0, 0.0, 0.0]`

### 右 base
- 位置：`[-0.273641, -0.285920, 0.770517]`
- 四元数（wxyz）：`[0.999961, 0.006516, 0.003163, -0.005038]`

### Head camera
- 位置：`[-0.142118, 0.016533, 1.214396]`
- 四元数（wxyz）：`[0.854012, 0.012553, 0.518857, -0.035978]`

## 这个 overview 视角下的轴解释

在 version B 当前 overview 设定里：

- 画面远处大致对应 world `+x`
- 画面右侧大致对应 world `-y`

所以它和 version A 里的 overview 不是同一个轴解释。
