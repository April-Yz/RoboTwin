# RoboTwin 功能-指令对照表

这个文档把 `code_painting/` 里常用流程整理成“功能名 -> 指令”。

## 1. 在 RoboTwin 里直接 replay 人手轨迹

用途：
- 回放 `hand_detections_<id>.npz`
- 检查重定向后的机械臂运动、相机视角和 viewer

主入口：
- `code_painting/run_hand_retarget_r1_npz.sh`
- `code_painting/run_hand_retarget_r1_npz_urdfik.sh`

示例：
```bash
bash /home/zaijia001/ssd/RoboTwin/code_painting/run_hand_retarget_r1_npz_urdfik.sh \
  /home/zaijia001/ssd/data/R1/gt_depth_vis/d_pour_blue/hand_vis/hand_detections_1.npz \
  /home/zaijia001/ssd/RoboTwin/code_painting/hand_replay_out \
  --enable_viewer 1
```

## 2. 在 RoboTwin 里 replay 单个 FoundationPose 物体轨迹

用途：
- 回放上游 `poses.npz` 的单物体轨迹

主入口：
- `code_painting/render_object_pose_r1_npz.py`

示例：
```bash
python /home/zaijia001/ssd/RoboTwin/code_painting/render_object_pose_r1_npz.py \
  --input_npz /path/to/poses.npz \
  --mesh_file /path/to/bottle.obj \
  --output_dir /home/zaijia001/ssd/RoboTwin/code_painting/object_replay_out \
  --enable_viewer 1
```

## 4. 在 RoboTwin 里执行 AnyGrasp 关键帧规划

用途：
- 读取 AnyGrasp `grasp_*.json`
- 按手的朝向去匹配候选
- 在 RoboTwin 里执行关键帧抓取/动作

主入口：
- `code_painting/run_plan_anygrasp_keyframes_r1_batch.sh`

支持：
- 双手同步执行
- `urdfik` 后端
- 手工指定 candidate
- 没到位前持续 replanning

示例：
```bash
CUDA_VISIBLE_DEVICES=3 bash /home/zaijia001/ssd/RoboTwin/code_painting/run_plan_anygrasp_keyframes_r1_batch.sh \
  /home/zaijia001/ssd/RoboTwin/code_painting/anygrasp_batch_results \
  /home/zaijia001/ssd/RoboTwin/code_painting/replay_m_obj_pose_d_pour_blue \
  /home/zaijia001/ssd/data/R1/gt_depth_vis/d_pour_blue/hand_vis \
  /home/zaijia001/ssd/RoboTwin/code_painting/anygrasp_plan_keyframes \
  --ids 1 \
  --keyframes 1 21 \
  --arm auto \
  --execute_both_arms 1 \
  --planner_backend urdfik \
  --manual_candidate 1 left 5 \
  --manual_candidate 1 right 11 \
  --replan_until_reached 1 \
  --replan_until_reached_max_attempts 20 \
  --reach_error_pose_source ee \
  --init_prefix_frames 15 \
  --pause_after_keyframe1_seconds 5 \
  --enable_viewer 1
```

关键参数：
- `--keyframes a b`：第一个关键帧和第二个关键帧
- `--execute_both_arms 1`：双手同步执行；每个 stage 必须两手都结束后才进入下一个 stage
- `--planner_backend urdfik|curobo`：后端 A 直接 IK，或者后端 B 的 cuRobo MotionGen 全轨迹规划
- `--urdfik_trajectory_mode joint_interp|cartesian_interp_ik`：在后端 A 内部，选择原来的“单终点 IK + 关节插值”模式，或者新的“末端笛卡尔插值 + 逐点 IK”模式
- `--urdfik_cartesian_interp_steps N`：当 `--urdfik_trajectory_mode cartesian_interp_ik` 时，指定 TCP 笛卡尔插值的 waypoint 数
- `--manual_candidate FRAME ARM IDX`：手工指定某一帧某只手的 candidate
- `--reach_error_pose_source ee|tcp`：误差按 wrist/endlink 还是 fingertip TCP 算
- `--replan_until_reached 1`：没到位就继续从当前位置规划
- `--init_prefix_frames N`：正式动作前先固定输出 N 帧初始姿态
- `--pause_after_keyframe1_seconds 5`：到达第一个关键帧并完成夹爪闭合后，保持当前姿态 5 秒，再开始第二个关键帧的规划和执行
- `--replay_objects_during_action 1`：在 `action` 阶段按 `replay_dir/multi_object_world_poses.npz` 回放物体轨迹，而不是让被选中的物体跟 TCP 走
- `--replay_objects_ignore_collision 1`：把 replay 物体作为 visual-only kinematic actor 加入场景，忽略与机械臂/其他物体的碰撞
- `--object_mesh_override NAME=/abs/path/to/mesh.obj`：覆盖某个 replay 物体的显示 mesh，例如把坏掉的 `cup.obj` 换成 `blue_cup.obj`
- `--candidate_keep_camera_up 1`：只解决夹爪绕前进轴“上下颠倒”的冗余，不强行固定到某个精确 roll；总体上让夹爪顶部 / 相机保持在上侧
- `--candidate_camera_top_axis z|y`：指定哪个局部轴当作“夹爪顶部轴”；当前建议先用 `z`
- `--candidate_target_local_x_offset_m VALUE`：在可视化和规划前，先沿夹爪局部 `+X` 平移每个 AnyGrasp target pose。如果当前 raw candidate 更像 wrist/endlink pose，而你想让 planner 输入改成指尖 TCP target，就用 `-0.12`

物体 replay 版示例：
```bash
CUDA_VISIBLE_DEVICES=3 bash /home/zaijia001/ssd/RoboTwin/code_painting/run_plan_anygrasp_keyframes_r1_batch.sh \
  /home/zaijia001/ssd/RoboTwin/code_painting/anygrasp_batch_results \
  /home/zaijia001/ssd/RoboTwin/code_painting/replay_m_obj_pose_d_pour_blue \
  /home/zaijia001/ssd/data/R1/gt_depth_vis/d_pour_blue/hand_vis \
  /home/zaijia001/ssd/RoboTwin/code_painting/anygrasp_plan_keyframes \
  --ids 1 \
  --keyframes 1 22 \
  --arm auto \
  --execute_both_arms 1 \
  --lighting_mode front_no_shadow \
  --planner_backend urdfik \
  --enforce_target_object_constraint 0 \
  --enforce_candidate_distance_constraint 0 \
  --debug_show_all_candidates 1 \
  --debug_common_candidate_top_k 0 \
  --debug_candidate_top_k 1 \
  --candidate_orientation_remap_label identity \
  --candidate_post_rot_xyz_deg 0 0 0 \
  --candidate_keep_camera_up 1 \
  --candidate_camera_top_axis z \
  --candidate_target_local_x_offset_m -0.12 \
  --manual_candidate 1 left 5 \
  --manual_candidate 1 right 8 \
  --manual_candidate 22 left 7 \
  --manual_candidate 22 right 1 \
  --replan_until_reached 1 \
  --replan_until_reached_max_attempts 20 \
  --reach_error_pose_source ee \
  --save_debug_preview 1 \
  --save_debug_execution_preview 1 \
  --enable_viewer 1 \
  --viewer_frame_delay 0.02 \
  --viewer_wait_at_end 1 \
  --init_prefix_frames 15 \
  --pause_after_keyframe1_seconds 5 \
  --execute_interp_steps 2 \
  --object_mesh_override cup=/home/zaijia001/ssd/data/R1/hand/obj_mesh/blue_cup/blue_cup.obj \
  --replay_objects_during_action 1 \
  --replay_objects_ignore_collision 1
```

说明：
- 当前实现只在第二段 `action` 中回放物体轨迹。
- 第一段 `pregrasp / grasp / pause_after_keyframe1` 里，物体仍固定在关键帧 1 的 pose，避免第一目标在抓取前漂走。
- 开启 `--replay_objects_during_action 1` 后，物体不会再跟随 TCP 附着移动；而是直接按 `multi_object_world_poses.npz` 的轨迹播放。
- 如果要对比“导出 world pose 时的 camera”和“planner 当前 camera”，需要先用新版 `render_multi_object_pose_r1_npz.py` 重新生成一次 `multi_object_world_poses.npz`，因为现在这个文件里会额外保存 `head_camera_pose_world_wxyz`。
- planner 侧可加：
  - `--save_pose_debug 1`
  这样会在 `output_dir/pose_debug.jsonl` 里同时记录：
  - 当前 `head_camera_pose_world_wxyz`
  - 当前左右 TCP
  - 当前物体 actor pose
  - replay 导出时同一帧的 `head_camera_pose_world_wxyz`
- planner 现在还会在执行预览阶段额外写：
  - `output_dir/debug_execution_metrics.jsonl`
  这是比 `pose_debug.jsonl` 更轻量的逐帧分析文件，重点记录：
  - 每只手的 target pose 和当前 pose
  - 物体计划位置和物体实际位置
  - `target-object`、`current-object` 的 xyz 误差和绝对距离
- planner 现在还会在 `output_dir/analysis_plots/` 下自动导出 PNG 图表：
  - `{left,right}_target_current_error_vs_time.png`
  - `{left,right}_object_distance_vs_time.png`
  这些图直接用时间做横轴，适合你现在这种“机械臂目标与真实位置之间差异随时间怎么变化”的分析。
- `debug_execution_preview.mp4` 现在会带一个彩色指标面板：
  - 黄色：阶段 / 目标物体
  - 青色：`target-current`
  - 红色：`obj_plan-obj_actual`
  - 橙色：`target-object`
  - 绿色：`current-object`
- 如果 keyframe 1 和 keyframe 22 主要只差一个绕夹爪前进轴的冗余 `360` 度自转，可以打开：
  - `--candidate_keep_camera_up 1`
  - `--candidate_camera_top_axis z`
  这样会保持局部前进轴不变，只消掉自由 roll，让夹爪上方的小相机尽量始终在上方。

## 5. 生成 RoboTwin 版 rank preview 图片

用途：
- 保存关键帧左右手 top-ranked 候选静态图
- 即使 viewer 打不开，也能人工选 candidate

主入口：
- 内置在 `code_painting/plan_anygrasp_keyframes_r1.py`

示例：
```bash
CUDA_VISIBLE_DEVICES=3 bash /home/zaijia001/ssd/RoboTwin/code_painting/run_plan_anygrasp_keyframes_r1_batch.sh \
  /home/zaijia001/ssd/RoboTwin/code_painting/anygrasp_batch_results \
  /home/zaijia001/ssd/RoboTwin/code_painting/replay_m_obj_pose_d_pour_blue \
  /home/zaijia001/ssd/data/R1/gt_depth_vis/d_pour_blue/hand_vis \
  /home/zaijia001/ssd/RoboTwin/code_painting/anygrasp_plan_keyframes \
  --ids 1 \
  --keyframes 1 21 \
  --save_rank_preview_images 1 \
  --rank_preview_top_n 3 \
  --enable_viewer 0
```

## 6. 不用 RoboTwin，直接生成 AnyGrasp 原图编号预览

用途：
- 不打开 RoboTwin，也不进入 SAPIEN 模拟器
- 直接在 AnyGrasp 自己的 `vis/grasp_result_*.png` 上叠加编号
- 按 AnyGrasp 原始 score 从高到低排序
- 左手、右手分别出图

主入口：
- `code_painting/render_anygrasp_ranked_preview.py`

示例：
```bash
/home/zaijia001/ssd/miniconda3/envs/RoboTwin_bw/bin/python \
  /home/zaijia001/ssd/RoboTwin/code_painting/render_anygrasp_ranked_preview.py \
  --anygrasp_dir /home/zaijia001/ssd/RoboTwin/code_painting/anygrasp_batch_results/d_pour_blue_1 \
  --hand_npz /home/zaijia001/ssd/data/R1/gt_depth_vis/d_pour_blue/hand_vis/hand_detections_1.npz \
  --base_image_dir /home/zaijia001/ssd/RoboTwin/code_painting/replay_m_obj_pose_d_pour_blue/d_pour_blue_1/head_anygrasp_frames \
  --base_image_mode raw \
  --output_dir /home/zaijia001/ssd/RoboTwin/code_painting/anygrasp_direct_preview/d_pour_blue_1 \
  --frames 1 21 \
  --top_k 2 \
  --draw_grasp_boxes 1
```

输出：
- `frame_000001_left_score_rank.png`
- `frame_000001_right_score_rank.png`
- `frame_000001_left_right_score_rank.png`
- `summary.json`

说明：
- 如果你不想看到 AnyGrasp 原来那张 `vis` 图里密集的夹爪，可视化建议加：
  - `--base_image_dir .../head_anygrasp_frames`
  - `--base_image_mode raw`
- 左右手已经通过输出文件名区分，所以编号可以统一用黑色小字。
- `--draw_grasp_boxes 1` 会在原始 RGB 图上给每个显示的候选画一个简化夹爪线框。
