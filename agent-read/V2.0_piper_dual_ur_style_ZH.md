# V2.0 Piper 双臂 UR 风格配置

## 目标

在完全不修改 R1 / R1 Pro 文件的前提下，提供一个真正面向 Piper 的 AnyGrasp 规划入口。

## 核心思路

这一版 V2 仿照项目里已有的 UR 结构：

- 使用 `dual_arm_embodied=false`
- 加载 **两个独立的单臂 Piper URDF**
- 用 `embodiment_dis` 拉开左右间距
- 在 replay 和 URDF-IK 阶段都保留 left/right 各自独立的 base pose

这和前一版兼容包装层不同。前一版仍然复用了太多 R1 风格的 replay 逻辑。

- `robot_config_Piper_dual_v2.json`

## 机器人布局

配置在 `robot_config_Piper_dual_v2.json` 中：

- `left_robot_file = ./assets/embodiments/piper`
- `right_robot_file = ./assets/embodiments/piper`
- `dual_arm_embodied = false`
- `embodiment_dis = 0.80`
- 左右拆分前的基础 pose：
  - `[0.0, -0.65, 0.72, 1.0, 0.0, 0.0, 0.0]`
- 经过 `Robot(...)` 的 UR 风格左右拆分后，实际 base pose 为：
  - left: `[-0.4, -0.65, 0.72]`
  - right: `[+0.4, -0.65, 0.72]`

所以 V2 实际上是两个固定在世界坐标中的桌面型 Piper，整体思路和 UR 的双单臂拼接方案一致。

## Renderer 行为

`replay_piper_dual_h5.py` 修复了前一版包装层的关键问题：

- 不会再把两个机械臂强行压回同一个 root pose
- 会显式保存：
  - `_left_base_pose`
  - `_right_base_pose`
- 提供按 arm 区分的 world/base 转换：
  - `world_pose_to_base_pose_for_arm(...)`
- head camera 不再依赖 R1 的 head link 语义，而是使用固定俯视相机位姿

## URDF IK 行为

`render_hand_retarget_piper_dual_npz_urdfik.py` 替换了 R1 专用的 URDFIK 假设：

- URDF：`assets/embodiments/piper/piper.urdf`
- base link：`base_link`
- ee link：`link6`
- 按 6 自由度机械臂处理 seed / IK 结果
- 不再带 R1 torso joints 假设
- Cartesian waypoint IK 在每个 arm 自己的 base frame 下进行
