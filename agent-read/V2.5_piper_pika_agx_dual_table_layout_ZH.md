# V2.5 Piper+Pika（agx 彩色版）双臂桌边布局

## 目标

使用新的彩色 Piper+Pika 组合，按照前面 UR 风格的双单臂思路，放到一张桌子的长边外侧。

## 桌子几何

按你的要求使用：

- 高：`0.75 m`
- 宽：`0.60 m`
- 长：`1.20 m`

在预览场景中，桌子被建模为一个简单长方体：

- 中心：`[0.0, 0.0, 0.375]`
- half size：`[0.60, 0.30, 0.375]`

因此桌面平面位于：

- `z = 0.75`

## 本轮采用的布局假设

为了保持和前面 UR 风格双单臂一致，这里采用了对称布局。

分裂前共享 pose：

- 之前错误版本：`robot_pose = [0.0, -0.60, 0.75, 1.0, 0.0, 0.0, 0.0]`
- 修正朝向但仍在桌外的版本：`robot_pose = [0.0, -0.60, 0.75, 0.70710678, 0.0, 0.0, 0.70710678]`
- 当前桌边安装版本：`robot_pose = [0.0, -0.30, 0.75, 0.70710678, 0.0, 0.0, 0.70710678]`

左右分离距离：

- `embodiment_dis = 0.60`

经过 `Robot(...)` 自动做左右平移后，得到实际 base：

- 左臂 base：`[-0.30, -0.60, 0.75]`
- 右臂 base：`[+0.30, -0.60, 0.75]`

含义是：

- 两个机械臂都放在同一条桌子长边外侧
- 桌子这条长边位于 `y = -0.30`
- 当前两个机械臂 base 位于 `y = -0.30`
- 因而当前版本是直接固定在桌子长边边缘，而不是悬在桌子外侧
- 左侧 base 距离桌子左短边 `0.30 m`
- 右侧 base 距离桌子右短边 `0.30 m`
- 因而两个 base 之间的间距正好是 `0.60 m`
- 在四元数修正后，两个机械臂的正前方都朝向 `+y`，也就是从桌子长边外侧朝桌面内部伸入
- 这才符合“夹爪从桌子长边一侧操作桌面物体”的目标

## 相机说明

之前那版斜视角预览本质上更像第三视角，而且画面方向有些反了，会让人感觉天花板和地板关系不对。为便于临时检查，我新增了一种俯视相机模式。

俯视相机放置方式：

- 位于两个机械臂 base 中点的正上方
- 世界坐标：`[0.0, -0.60, 2.10]`
- 朝向：直接向下看桌面

参考现有 UR 风格回放逻辑：

- `code_painting/replay_piper_dual_h5.py` 使用一个固定 overview/head camera fallback：
  - position: `[-0.032, -0.45, 1.35]`
  - forward: `[0.0, 0.6, -0.8]`
  - left: `[-1.0, 0.0, 0.0]`
- `code_painting/render_hand_retarget_r1_npz.py` 里的第三视角是根据机器人前向和世界上方向自动构造的，而不是随意放一个可能翻转的相机

基于这个参考，本轮把 oblique 视角改成了位于机械臂后方、朝桌面看的正常斜视角。

现在脚本支持：

- `--camera-mode top_down`
- `--camera-mode oblique`
