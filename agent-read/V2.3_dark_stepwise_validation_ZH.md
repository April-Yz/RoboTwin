# V2.3 Piper / Pika / 组合模型的暗灯光分步验证

## 目标

在较暗灯光下分步验证外观：

1. 原始 Piper 手臂单独预览
2. 原始 Pika gripper 单独预览
3. 组合后的 `piper_pika` 模型预览

## 主要结论

组合模型发白，不只是灯光问题。

目前已经定位到一个结构差异：

- 原始 Piper 手臂 URDF：
  - visual 使用 DAE
  - 对手臂 link 本身**没有**再额外定义 URDF `<material><color ...>`
  - 所以 SAPIEN 更可能直接使用 DAE 内嵌的材质/effect 信息
- 组合后的 `piper_pika.urdf`：
  - 虽然手臂 visual 已切回 DAE
  - 但仍然保留了从旧组合参考继承下来的显式浅色 URDF material，例如：
    - `rgba="0.792156862745098 0.819607843137255 0.933333333333333 1"`
  - 这很可能在 SAPIEN 中覆盖或冲淡了 DAE 里的原始材质外观

所以当前最合理的解释是：

- 原始 Piper 手臂可以显示出更接近目标的深灰色
- 组合模型里的 Piper 手臂之所以发白，是因为组合 URDF 里仍然保留了显式的浅色 material 覆盖
