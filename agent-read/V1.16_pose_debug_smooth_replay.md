# V1.16 Smooth replay from pose_debug.jsonl

## Purpose

When `plan_anygrasp_keyframes_r1.py` runs with:

- `--pure_scene_output 1`

it keeps the main export clean and automatically writes:

- `pose_debug.jsonl`

This file records per-frame:

- left/right arm joint positions
- gripper command and gripper joint qpos
- head camera / TCP / EE poses
- current execution-object actor poses

A new post-processing replay tool is now available:

- `code_painting/replay_pose_debug_smooth.py`
- wrapper: `code_painting/run_replay_pose_debug_smooth.sh`

Its role is:

- do not change the original planning/execution result
- re-render the recorded pose-debug trajectory with denser interpolation
- export a visually smoother replay video

## Core semantics

The new path does:

1. load `pose_debug.jsonl`
2. interpolate between adjacent records:
   - arm joint qpos: linear interpolation
   - gripper joint qpos / gripper command: linear interpolation
   - object actor poses: linear position interpolation + quaternion slerp
3. render the interpolated states into a new video

So this is:

- **post-hoc visual replay smoothing**

not:

- online control compensation
- replanning
- improving the original reach accuracy

## Important limitation

This tool improves:

- video appearance
- presentation smoothness

It does **not** change:

- the original run's reach result
- the original run's `plan_summary.json` errors
- the original controller convergence behavior

In short:

- it makes the replay video smoother
- it does not make the original execution more accurate

## Recommended use case

Use it when:

- a pure-mode run already finished
- you want a smoother presentation video from that exact run
- you do not want to re-run AnyGrasp / planning / execution

Do not use it as a substitute for:

- controller-follow analysis
- reach / attempt / pos_err / rot_err debugging

## Common commands

### 1. Infer everything from `plan_summary.json`

```bash
bash /home/zaijia001/ssd/RoboTwin/code_painting/run_replay_pose_debug_smooth.sh \
  --plan_summary_json /path/to/d_pour_blue_35/plan_summary.json \
  --output_path /path/to/d_pour_blue_35/head_cam_plan_smooth.mp4 \
  --interp_factor 4 \
  --fps 20 \
  --overlay_text 0 \
  --base_occluder_enable 1 \
  --base_occluder_local_pos 0.0 0.0 0.4 \
  --base_occluder_half_size 0.45 0.45 0.02 \
  --base_occluder_color 1.0 1.0 1.0
```

This automatically reads:

- `pose_debug.jsonl`
- `replay_dir`

from the summary.

### 2. Specify `pose_debug.jsonl` directly

```bash
bash /home/zaijia001/ssd/RoboTwin/code_painting/run_replay_pose_debug_smooth.sh \
  --pose_debug_jsonl /path/to/d_pour_blue_35/pose_debug.jsonl \
  --replay_dir /path/to/replay_m_obj_pose_d_pour_blue_norobot/d_pour_blue_35 \
  --output_path /path/to/d_pour_blue_35/head_cam_plan_smooth.mp4 \
  --interp_factor 4 \
  --fps 20
```

## Key parameters

- `--plan_summary_json`
  - recommended entry; infers `pose_debug.jsonl` and `replay_dir`
- `--pose_debug_jsonl`
  - direct replay input
- `--replay_dir`
  - used to load replay object meshes
- `--output_path`
  - output video path
- `--interp_factor`
  - number of replay subframes inserted per adjacent recorded pair
  - `1` means no extra interpolation
  - recommended starting value: `3` or `4`
- `--fps`
  - output video FPS
- `--overlay_text`
  - whether to draw lightweight labels onto the smoothed replay video
- `--max_input_records`
  - replay only the first N records; useful for smoke tests

## Recommended settings

For a natural-looking but not overly slow replay:

- `--interp_factor 4`
- `--fps 20`

For quick pipeline checking:

- `--interp_factor 2`
- `--max_input_records 20`

## Relation to execution tuning

This replay tool is best used in the following workflow:

1. tune real execution parameters first
   - `joint_command_scene_steps`
   - `settle_steps`
   - `joint_target_wait_steps`
2. confirm real `reached / pos_err / rot_err`
3. then use `pose_debug.jsonl` for post-hoc presentation smoothing

Recommended principle:

- solve **real execution performance** in execution parameters
- solve **video appearance** with `replay_pose_debug_smooth.py`
