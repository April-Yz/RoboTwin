# V2.6 Calibrated Piper+Pika Scene Reconstruction

## Goal

Reconstruct a simulated scene using the real-world calibration bundle so that:

- the first Piper+Pika robot follows the manually tuned tabletop placement
- the second Piper+Pika robot is placed from the calibrated base-to-base transform
- the head camera is placed from the calibrated head-camera transform
- all outputs live under `code_painting/pika/`

## Calibration inputs used

Read from:

- `CALIBRATION_TRANSFORMS_README.md`
- `calibration_bundle_try2.json`

Transforms used:

- `left_base_T_right_base`
- `left_base_T_head_camera`

Interpretation:

- the simulated anchor robot is treated as `left_base`
- the second robot pose is computed as:
  - `world_T_right_base = world_T_left_base * left_base_T_right_base`
- the head camera pose is computed as:
  - `world_T_head_camera = world_T_left_base * left_base_T_head_camera`

## Anchor robot placement

The first robot uses the current manual tabletop placement from:

- `robot_config_PiperPika_agx_dual_table.json`

Current shared pose in that config:

- `robot_pose = [0.0, -0.25, 0.75, 0.70710678, 0.0, 0.0, 0.70710678]`

With `embodiment_dis = 0.60`, the left-arm anchor base becomes:

- left base world pose: `[-0.30, -0.25, 0.75]`

This anchor is used as the simulated `left_base`.

## Implementation

Added script:

- `code_painting/pika/visualize_calibrated_piper_pika_scene.py`

What it does:

- loads two `piper_pika_agx.urdf` robots manually
- places the first robot from the manual config-derived `left_base`
- places the second robot from `left_base_T_right_base`
- adds a simple head-camera marker actor
- renders:
  - one overview image
  - one overview video
  - one image from the calibrated head-camera pose

## Notes

- This reconstruction uses the current manual placement as the simulated `left_base` anchor.
- If the manual tabletop placement is adjusted later, rerunning the script will automatically move the calibrated right robot and head camera together with the anchor.
- The head camera is calibrated in the D435 color optical frame. The script converts that optical-frame orientation into a SAPIEN render camera pose using:
  - forward = optical `z`
  - left = `-optical x`
