# V1.14 Preview Top1 执行关系说明

这份说明专门记录当前基于 preview `summary.json` 的 AnyGrasp planner 执行逻辑。

相关代码：
- `code_painting/plan_anygrasp_keyframes_r1.py`
- `code_painting/plan_anygrasp_keyframes_r1_batch.py`

## 1. 候选来源

这个模式下，planner 在关键帧选择阶段不会重新扫描所有 AnyGrasp 候选。

它会直接读取 preview 输出 JSON：
- `render_anygrasp_ranked_preview.py` 产出的 `summary.json`
- 每只手对应的 top 列表：
  - `top_candidates.left_orientation`
  - `top_candidates.right_orientation`
  - 如果显式指定，也可以读 fused 列表

当参数是：
- `--reuse_preview_candidate_group orientation`
- `--reuse_preview_top_rank 1`

planner 实际读取的是：
- `top_candidates.left_orientation[0]`
- `top_candidates.right_orientation[0]`

也就是每帧每只手的 top1。

## 2. 关键帧规则

如果命令里使用：
- `--keyframes 1 22`
- `--candidate_selection_relative_frame -10`

那么关键帧解析以 preview 的 `resolved_frames` 为准，最终执行帧是：
- 关键帧1 = `1` 对应解析后的帧
- 关键帧2 = `max(resolved(22), resolved(-10))`

对于当前这组 `fi60` preview：
- `resolved(-10) = 28`
- 所以最终两帧是 `1` 和 `28`

## 3. 阶段顺序

当前执行顺序是：
1. 先到 `pregrasp`
2. 再从 `pregrasp` 到关键帧1抓取位姿
3. 闭合夹爪
4. 再移动到关键帧2动作位姿

这里的 `pregrasp` 不是关键帧1本身。

它是用关键帧1抓取位姿，沿夹爪局部前进轴反向回退得到的：
- 偏移量 = `--approach_offset_m`
- 当前默认值 = `0.08` 米

因此当前行为实际就是：
- 先把 EE/TCP 平滑移动到 `pregrasp`
- 再从 `pregrasp` 短距离推进到关键帧1抓取位姿
- 闭合夹爪
- 再把 EE/TCP 平滑移动到关键帧2

对于推荐配置：
- `--planner_backend urdfik`
- `--urdfik_trajectory_mode cartesian_interp_ik`

平滑方式是先在 EE/TCP 笛卡尔空间插值，再逐路点做 IK。

## 4. 物体与夹爪的关系

在候选选择阶段，夹爪不会自动和物体建立“抓住了”的关系。

选择阶段：
- 夹爪位姿来自 AnyGrasp preview 的 top1
- 物体名称主要用于筛选和标注
- 这时还没有物理夹持或刚性附着关系

闭合前执行阶段：
- 在 `pregrasp` 和 `grasp` 阶段，物体保持在 replay 的关键帧1世界系位姿
- 夹爪独立朝着选中的 AnyGrasp 位姿移动

闭合后执行阶段：
- 如果 `--replay_objects_during_action 0`：
  - 选中的物体会在 `action` 阶段刚性附着到当前执行臂的 TCP
  - 没被选中的物体保持在固定 replay 位姿
- 如果 `--replay_objects_during_action 1`：
  - 所有物体按 `multi_object_world_poses.npz` 里的轨迹播放
  - 不会附着到 TCP

单臂默认模式：
- 只有一只手执行
- 只有这只手选中的物体会在 `action` 阶段有机会附着到 TCP

双臂同步模式：
- 如果双臂都执行，且关闭 object replay
- 那么左右手各自选中的物体会在闭合后分别附着到对应 TCP

## 5. 有头 Viewer 模式

这条 preview-top1 流程也支持有头界面运行。

常用参数：
- `--enable_viewer 1`
- `--viewer_frame_delay 0.02`
- `--viewer_wait_at_end 1`

这些参数不会改变选择逻辑，只是把同一套执行流程显示到交互窗口里。

## 7. 关键参数怎么调

路径相关：
- `anygrasp_batch_results`
  - 原始 AnyGrasp 结果根目录
- `replay_m_obj_pose_d_pour_blue_norobot`
  - replay 物体轨迹根目录
- `hand_vis`
  - `hand_detections_<id>.npz` 所在目录
- `anygrasp_plan_keyframes_from_preview_fi60`
  - 本次 planner 输出目录
- `--reuse_preview_summary_root`
  - preview 批处理结果根目录，planner 会自动到每个子目录下找 `summary.json`

候选来源相关：
- `--reuse_preview_candidate_group orientation|fused`
  - `orientation`：按 hand orientation 排序后的 top 列表取候选
  - `fused`：按 AnyGrasp 分数和 orientation 分数融合后的列表取候选
- `--reuse_preview_top_rank 1`
  - 取 top1
  - 改成 `2` 就是取 top2

关键帧相关：
- `--keyframes 1 22`
  - 基础两帧
- `--candidate_selection_relative_frame -10`
  - 再引入相对帧
  - 最终第二帧是 `max(22, resolved(-10))`
- `--candidate_selection_mode top_score_auto`
  - 在 preview-top1 这条路径里，它主要用来保持“auto 选手时按分数优先”的语义一致

运动方式相关：
- `--planner_backend urdfik`
  - 当前推荐
- `--urdfik_trajectory_mode cartesian_interp_ik`
  - 先走 EE/TCP 笛卡尔平滑，再逐点 IK
- `--urdfik_cartesian_interp_steps 30`
  - 越大越平滑，速度越慢
- `--approach_offset_m 0.08`
  - 关键帧1前退 8cm 作为 `pregrasp`

执行模式相关：
- `--arm auto`
  - 左右手都先读各自 top1，再决定执行哪一只手
- `--arm left|right`
  - 强制只执行某一只手
- `--execute_both_arms 1`
  - 双臂同步执行
- `--execute_both_arms 0`
  - 单臂执行，当前更稳

物体关系相关：
- `--replay_objects_during_action 0`
  - 闭合后，选中物体附着到 TCP
- `--replay_objects_during_action 1`
  - 物体继续按 replay 轨迹走，不附着 TCP

调试和显示相关：
- `--save_debug_preview 1`
  - 保存关键帧选择预览视频
- `--save_debug_execution_preview 1`
  - 保存慢速执行预览视频
- `--enable_viewer 1`
  - 打开有头界面
- `--viewer_frame_delay 0.02`
  - viewer 播放速度
- `--viewer_wait_at_end 1`
  - 结束后不立刻退出
