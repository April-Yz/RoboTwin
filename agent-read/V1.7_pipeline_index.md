# V1.7 Pipeline Index

## What This Pipeline Covers

The current `code_painting/` workflow connects real-robot perception outputs to RoboTwin replay and then to grasp execution debugging.

There are five stages:

1. Upstream hand extraction
2. RoboTwin hand replay
3. Upstream FoundationPose extraction
4. RoboTwin object replay
5. RoboTwin AnyGrasp candidate selection and execution

## One-Line Flow

- `hand_detections_<id>.npz` -> replay human-hand targets in RoboTwin
- `poses.npz` -> replay object trajectories in RoboTwin
- replay RGB-D export -> run AnyGrasp elsewhere
- AnyGrasp JSON + replayed object world poses + hand NPZ -> choose grasp candidates and execute keyframe plan in RoboTwin

## Practical Order

1. Obtain `hand_detections_<id>.npz`
2. Replay hand trajectory in RoboTwin
3. Obtain FoundationPose `poses.npz`
4. Replay object trajectory and export AnyGrasp RGB-D
5. Run AnyGrasp on another server
6. Feed AnyGrasp results back into RoboTwin keyframe planner
