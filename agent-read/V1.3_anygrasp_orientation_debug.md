# V1.3 AnyGrasp Orientation Debug

## Summary
A comparison between the hand replay pipeline and the AnyGrasp planner pipeline showed that the AnyGrasp path likely missed a fixed local orientation conversion layer.

## Evidence
### Hand replay
- Uses `remap_target_rotation()`
- Applies `stored_orientation_post_rot_matrix`
- Applies `orientation_remap_matrix`

### AnyGrasp planner before this update
- Used the raw AnyGrasp `rotation_matrix` directly in `camera_to_world_pose()`
- Did not expose an equivalent fixed post-rotation or orientation remap stage

## Code changes
- Added `--candidate_orientation_remap_label`
- Added `--candidate_post_rot_xyz_deg`
- Applied these transforms before `camera_to_world_pose()` in the AnyGrasp candidate pipeline
- Updated debug gripper actors to visualize AnyGrasp `width` so the opening is visible in the viewer

## Recommended debug mode
- `--debug_candidate_top_k 1`
- `--debug_common_candidate_top_k 0`
- Then sweep a few fixed candidate post-rotations or remap labels
