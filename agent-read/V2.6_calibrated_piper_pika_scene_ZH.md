# V2.6 基于标定结果的 Piper+Pika 场景重建

## 目标

根据真实场景标定结果在模拟器里重建一个场景，使得：

- 第一台 Piper+Pika 机械臂沿用手动调好的桌面摆放位置
- 第二台 Piper+Pika 机械臂使用左右 base 标定外参来放置
- head cam 使用标定得到的 head 外参来放置
- 所有输出都放到 `code_painting/pika/` 下

## 使用的标定输入

读取：

- `CALIBRATION_TRANSFORMS_README.md`
- `calibration_bundle_try2.json`

使用的关键变换：

- `left_base_T_right_base`
- `left_base_T_head_camera`

解释：

- 仿真中的锚定机械臂被视为 `left_base`
- 第二台机械臂世界位姿通过下式得到：
  - `world_T_right_base = world_T_left_base * left_base_T_right_base`
- head cam 世界位姿通过下式得到：
  - `world_T_head_camera = world_T_left_base * left_base_T_head_camera`

## 第一台机械臂的锚定位姿

第一台机械臂直接使用当前手动调好的桌面布局配置：

- `robot_config_PiperPika_agx_dual_table.json`

当前该配置中的共享 pose 为：

- `robot_pose = [0.0, -0.25, 0.75, 0.70710678, 0.0, 0.0, 0.70710678]`

结合 `embodiment_dis = 0.60`，得到锚定左臂 base：

- left base world pose: `[-0.30, -0.25, 0.75]`

这个锚定姿态被用作仿真中的 `left_base`。

## 根据标定导出的世界位姿

验证运行打印得到：

### 左 base
- 位置：`[-0.30, -0.25, 0.75]`
- 四元数（wxyz）：`[0.707107, 0, 0, 0.707107]`

### 右 base
- 位置：`[0.271841, -0.273641, 0.770517]`
- 四元数（wxyz）：`[0.710642, 0.002371, 0.006844, 0.703517]`

### Head camera
- 位置：`[-0.0306125, -0.142118, 1.214396]`
- 四元数（wxyz）：`[0.629318, -0.358011, 0.375763, 0.578437]`

## 实现

新增脚本：

- `code_painting/pika/visualize_calibrated_piper_pika_scene.py`

它的作用：

- 手动加载两个 `piper_pika_agx.urdf`
- 第一台机械臂使用手动配置导出的 `left_base`
- 第二台机械臂使用 `left_base_T_right_base`
- 加一个简单的 head camera marker actor
- 输出：
  - 一张总览图
  - 一段总览视频
  - 一张从标定 head cam 视角拍到的图

## 说明

- 这次重建把当前手动摆好的第一台机械臂视为仿真的 `left_base` 锚点。
- 如果你后续继续改 `robot_config_PiperPika_agx_dual_table.json` 里的手动位置，只需要重新运行这个脚本，第二台机械臂和 head cam 会自动跟着新的锚点一起更新。
- head camera 的标定坐标系是 D435 color optical frame。脚本里把这个 optical frame 转成了 SAPIEN 渲染相机姿态，转换规则是：
  - forward = optical `z`
  - left = `-optical x`
