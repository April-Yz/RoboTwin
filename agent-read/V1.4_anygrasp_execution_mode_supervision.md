# V1.4 AnyGrasp Execution-Mode Clarification

Date: 2026-03-17

## What changed
- In `code_painting/plan_anygrasp_keyframes_r1.py`, execution-mode metadata is now explicit:
  - Added summary field `executed_arms`.
  - Added summary field `supervision_only_arms`.
- Added runtime log line:
  - `[exec-mode] selected_arm=<arm> supervision_only_arms=<...>`

## Why
Single-arm execution was easy to misread from videos: users could see left-arm motion and assume right-arm non-arrival indicated planning failure. In this workflow, right arm can be a supervision target without being actuated.

## Affected workflow
- AnyGrasp keyframe planner stage execution (`pregrasp -> grasp -> action`) for R1.
- JSON result interpretation (`plan_summary.json`).

## Expected input/output behavior
Input:
- Existing arguments and manual candidates remain unchanged.

Output:
- Existing stage metrics remain unchanged.
- `plan_summary.json` now explicitly tells which arms were executed and which were supervision-only.

## Notes
- This change does not introduce dual-arm execution.
- To test true right-arm reachability, run with `--arm right` and matching right-side manual candidate(s).
