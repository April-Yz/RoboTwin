# V1.7 Hand Pipeline

## Scope

This document only covers the RoboTwin-side hand pipeline.

Boundary:
- `hand_detections_<id>.npz` is upstream input.
- RoboTwin replays it.
- RoboTwin does not currently generate it.

## Main Scripts

- Basic replay: [run_hand_retarget_r1_npz.sh](/home/zaijia001/ssd/RoboTwin/code_painting/run_hand_retarget_r1_npz.sh)
- URDFIK replay: [run_hand_retarget_r1_npz_urdfik.sh](/home/zaijia001/ssd/RoboTwin/code_painting/run_hand_retarget_r1_npz_urdfik.sh)
- Batch replay: [run_hand_retarget_r1_npz_urdfik_pool.sh](/home/zaijia001/ssd/RoboTwin/code_painting/run_hand_retarget_r1_npz_urdfik_pool.sh)

## Input And Output

Typical input path:
- `/home/zaijia001/ssd/data/R1/gt_depth_vis/<task>/hand_vis/hand_detections_<id>.npz`

Typical output root:
- `/home/zaijia001/ssd/RoboTwin/code_painting/output_hand_retarget_*`

## Logic

1. Read the hand NPZ.
2. Build left and/or right target poses from stored gripper pose or keypoints.
3. Convert those targets into RoboTwin coordinates.
4. Plan and replay robot motion.
5. Save video and debug artifacts.

## Key Parameters

### Input and frame selection

- `--input_npz`: one hand NPZ file
- `--frame_start`, `--frame_end`, `--frame_stride`, `--max_frames`: frame subset
- `--arms left|right|both`: active arms

### Target construction

- `--pose_source gripper|retreat`: choose target source
- `--require_stored_gripper_pose 1`: require precomputed gripper pose in NPZ
- `--retreat_distance`: retreat offset

### Orientation

- `--orientation_remap_label`: fixed orientation remap
- `--stored_orientation_post_rot_xyz_deg`: fixed Euler post-rotation
- `--debug_force_orientation`: replace incoming orientation for debugging

### Position correction

- `--target_world_offset_xyz`: global XYZ offset
- `--left_target_world_offset_xyz`: left-only XYZ offset
- `--right_target_world_offset_xyz`: right-only XYZ offset
- `--target_world_z_offset`: global Z offset

### Rendering

- `--enable_viewer`: open SAPIEN viewer
- `--viewer_wait_at_end`: keep viewer open
- `--debug_visualize_targets`: draw target axes
- `--lighting_mode default|front|front_no_shadow`: lighting preset

```bash
  --viewer_wait_at_end 1
```

## When To Use Which Script

- Use `run_hand_retarget_r1_npz.sh` for a quick replay.
- Use `run_hand_retarget_r1_npz_urdfik.sh` when you want the current preferred replay behavior.
- Use `run_hand_retarget_r1_npz_urdfik_pool.sh` when you need a whole dataset rendered with one setting.
