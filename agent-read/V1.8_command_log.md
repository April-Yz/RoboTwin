# RoboTwin Command Log

This file records the main user-facing features in `code_painting/` as "feature name -> command".

## 1. Replay Hand Trajectory In RoboTwin

Purpose:
- Replay `hand_detections_<id>.npz` in RoboTwin.
- Check retargeted arm motion, camera alignment, and viewer rendering.

Main entry:
- `code_painting/run_hand_retarget_r1_npz.sh`
- `code_painting/run_hand_retarget_r1_npz_urdfik.sh`

Example:
```bash
bash /home/zaijia001/ssd/RoboTwin/code_painting/run_hand_retarget_r1_npz_urdfik.sh \
  /home/zaijia001/ssd/data/R1/gt_depth_vis/d_pour_blue/hand_vis/hand_detections_1.npz \
  /home/zaijia001/ssd/RoboTwin/code_painting/hand_replay_out \
  --enable_viewer 1
```

## 4. Run AnyGrasp Keyframe Planner In RoboTwin

Purpose:
- Load AnyGrasp grasp JSONs.
- Match candidates against hand orientation.
- Replay selected grasp/action keyframes in RoboTwin.

Main entry:
- `code_painting/run_plan_anygrasp_keyframes_r1_batch.sh`

Feature:
- dual-arm synchronized execution
- `urdfik` backend
- optional manual candidate overrides
- stage replan-until-reached

Example:
```bash
CUDA_VISIBLE_DEVICES=3 bash /home/zaijia001/ssd/RoboTwin/code_painting/run_plan_anygrasp_keyframes_r1_batch.sh \
  /home/zaijia001/ssd/RoboTwin/code_painting/anygrasp_batch_results \
  /home/zaijia001/ssd/RoboTwin/code_painting/replay_m_obj_pose_d_pour_blue \
  /home/zaijia001/ssd/data/R1/gt_depth_vis/d_pour_blue/hand_vis \
  /home/zaijia001/ssd/RoboTwin/code_painting/anygrasp_plan_keyframes \
  --ids 1 \
  --keyframes 1 21 \
  --arm auto \
  --execute_both_arms 1 \
  --planner_backend urdfik \
  --manual_candidate 1 left 5 \
  --manual_candidate 1 right 11 \
  --replan_until_reached 1 \
  --replan_until_reached_max_attempts 20 \
  --reach_error_pose_source ee \
  --init_prefix_frames 15 \
  --enable_viewer 1
```

Key parameters:
- `--keyframes a b`: keyframe-1 grasp target and keyframe-2 action target
- `--execute_both_arms 1`: synchronize both arms; both must finish each stage before the next stage
- `--manual_candidate FRAME ARM IDX`: manually override one candidate
- `--reach_error_pose_source ee|tcp`: choose wrist/endlink or fingertip TCP for error computation
- `--replan_until_reached 1`: keep replanning from current state until tolerance or attempt limit
- `--init_prefix_frames N`: prepend fixed init frames before motion starts

## 6. Generate Raw AnyGrasp Annotated Preview Without RoboTwin

Purpose:
- Do not open RoboTwin or SAPIEN.
- Directly annotate AnyGrasp's own `vis/grasp_result_*.png`.
- Show `candidate_idx` labels on the original AnyGrasp image.
- Produce left and right previews separately, while sorting by raw AnyGrasp score.

Main entry:
- `code_painting/render_anygrasp_ranked_preview.py`

Example:
```bash
/home/zaijia001/ssd/miniconda3/envs/RoboTwin_bw/bin/python \
  /home/zaijia001/ssd/RoboTwin/code_painting/render_anygrasp_ranked_preview.py \
  --anygrasp_dir /home/zaijia001/ssd/RoboTwin/code_painting/anygrasp_batch_results/d_pour_blue_1 \
  --hand_npz /home/zaijia001/ssd/data/R1/gt_depth_vis/d_pour_blue/hand_vis/hand_detections_1.npz \
  --base_image_dir /home/zaijia001/ssd/RoboTwin/code_painting/replay_m_obj_pose_d_pour_blue/d_pour_blue_1/head_anygrasp_frames \
  --base_image_mode raw \
  --output_dir /home/zaijia001/ssd/RoboTwin/code_painting/anygrasp_direct_preview/d_pour_blue_1 \
  --frames 1 21 \
  --top_k 2 \
  --draw_grasp_boxes 1
```

Outputs:
- `frame_000001_left_score_rank.png`
- `frame_000001_right_score_rank.png`
- `frame_000001_left_right_score_rank.png`
- `summary.json`

Notes:
- `--base_image_dir .../head_anygrasp_frames --base_image_mode raw` is the recommended mode when you want a clean image without AnyGrasp's original dense gripper overlay.
- Left and right are separated by output file, so candidate numbers can stay black without color-coding.
- `--draw_grasp_boxes 1` draws a lightweight gripper wireframe for each displayed candidate on top of the raw RGB image.
